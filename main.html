  • session에서 로그인 정보를 가져와 로그인 유무 판별 후 메뉴 동적 생성.
  • 등록된 이벤트 이미지 정보를 DB에서 가져와 javascript로 슬라이드형태로 노출하여, 이벤트이미지에 해당하는 이벤트페이지로 이동하는 링크를 삽입.
  • Hit, New, Best 태그 정보에 따라 상품목록을 가져옴.
- 참조 DB
  • 물품리스트
    SELECT ... FROM View_productList WHERE SPECTAG=?
  • 이벤트 이미지 및 경로
    SELECT ... FROM EVENT WHERE STATUS=1
[002. 회원가입]
- 화면 설명
아이디, 아이디중복 확인 버튼, 비밀번호, 비밀번호 확인, 이름, 이메일, 연락처, 우편번호 찾기 버튼 및 주소, 생년월일을 입력할 수 있는 텍스트 박스가 있음
- 구현 기능
  • 아이디중복버튼 클릭 시 유저 입력값이 DB에 있는 지 비동기적으로 판별하여 결과를 우측에 노출 (Ajax 이용)
  • 비밀번호와 비밀번호 확인 입력값이 일치하는 지를 jQuery로 비교 (blur함수로 포커싱 아웃될 때 each함수로 문자열 비교)
  • 우편번호 버튼 클릭 시 다음 우편번호 찾기 API modal창이 오픈, 주소 검색 후 결과값을 javascript로 우편번호, 주소 입력박스에 자동 셋팅
  • datepicker를 이용하여 생년월일을 캘린더 형태로 선택/입력 가능
  • 이용약관은 iframe 태그로 노출되며, 동의 체크박스에 체크가 되어야만 회원가입 기능이 활성화됨
  • 모든 입력박스의 값이 정상적으로 입력됐을 경우, 비밀번호는 SHA256 암호화, 이메일과 연락처는 AES256 암호화 처리를 거쳐 DB에 insert함
  • [공통] jQurey blur 함수, 정규표현식을 이용한 유효성 검사
- 참조 DB
  • 아이디 중복 검사
    SELECT COUNT(*) FROM MEMBER WHERE USERID=?
  • 회원 정보 입력
    INSERT INTO MEMBER VALUES(...)
[003. 로그인]
- 화면 설명
1) 로그인 아이디와 비밀번호 입력창/아이디 저장 체크박스/아이디 찾기 및 비밀번호 찾기 버튼
2) ID 찾기 아이디찾기 버튼 클릭 시 유저명과 연락처를 입력하는 modal이 나타남
3) PW 찾기 비밀번호 찾기 버튼 클릭 시 아이디와 이메일을 입력하는 modal이 나타남
찾기 클릭시 메일이 전송되며 인증번호 입력 화면으로 전환
인증번호 확인 후 새 비밀번호 입력, 비밀번호 확인 입력 박스 노출
- 구현 기능
  • 유저가 입력한 ID와 PW가 DB와 일치하는지 비교
  • PW는 SHA256 암호화 하여 DB와 비교한 후 일치하면 session에 로그인 유저정보가 담긴 Bean을 저장
  • 아이디저장 체크박스를 선택한 경우 아이디가 cookie에 저장하여 다음 접속시 cookie값을 자동 셋팅
  • 유저가 입력한 이름과 AES256 암호화 한 연락처를 DB와 일치하는 지 비교
  • 일치하는 경우 하단에 유저의 ID를 노출하고, 불일치 하는 경우 경고 텍스트가 노출
  • 아이디와 AES256 암호화 한 이메일 주소를 DB와 일치하는 지 비교하여 일치하는 경우 java Random함수로 생성된 인증코드를 Gmail API를 이용하여 유저의 이메일로 전송함
  • java에서 생성된 인증코드와 유저가 입력한 인증코드가 일치 하는 경우 새 비밀번호 입력화면으로 전환
  • jQuery로 입력한 비밀번호와 비밀번호 확인 정보가 일치하는지 유효성검사를 거친 후 비밀번호가 변경, 마지막 비밀번호 변경 일시도 update됨
- 참조 DB
  • 로그인
    SELECT ... FROM MEMBER WHERE USERID=? AND PW=?
  • 아이디찾기
    SELECT ... FROM MEMBER WHERE NAME=? AND PHONE=?
  • 비밀번호찾기
    SELECT ... FROM MEMBER WHERE USERID=? AND EMAIL=?
